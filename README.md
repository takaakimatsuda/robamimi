# 概要

『ロバミミ』はネタバレを防止する事に特化した**スレッド型SNS**です。

投稿者がスレッドを立てることでコンテンツの感想をクローズドな環境で共有し合うことができます。

# 制作背景

友人が「**SNSでアニメのネタバレをくらった！**」という発言をきっかけに作成しました。

Twitterは仕様上、新しい情報を無造作に目にしてしまいます。ミュート機能などを駆使しても100%見たくない情報をシャットダウンすることは出来ません。

ネタバレを気にしてしまう人は情報を見てしまうリスクがある・ネタバレを気にしない人はタイムリーな情報を発信するのに気を遣う状況が出来てしまいます。

課題まとめ
* Twitterではネタバレ情報を見てしまうリスクがある。
* ネタバレの情報を発信したいがネタバレに気を遣って発信できない。
* ネタバレ情報を見た人間と見ていない人間でお互いに負の感情を抱いてしまう。

解決策

* 話題にしたい内容でスレッドを立てることで、**クローズドな環境でユーザーが意見交換できる。**

# 工夫した点

### TwitterAPIの使用

ユーザー登録機能を作成するにあたって、下記２点が課題にありました。

* 新規登録はユーザーからの心理的ハードルが高い
* AWSでユーザー画像をDB登録しているのでコストがかかる

TwitterAPIを利用してユーザー登録することでワンクリックで登録できる上にTwitterのアイコンがそのまま使えるようになりました。

### UI/UX

* サイトのデザインは「シンプルに一貫性をもたせる」というテーマで決めました。
* 既存のWebアプリのデザインを参考に(Slackなど)。

### 環境・技術

* チーム開発を意識して、Gitで新たな機能を実装する時はブランチを切ってからマージするようにしていました。

# 最後に

Laravelを利用したオリジナルアプリを初めて制作しました。
仕事と並行して挫折せずに完成までこぎつけたのは以下２点の要因が大きいかと思います。

1. 一番実現したい機能を実装し、他の部分はシンプルにしたこと

以前に作成したWebアプリは色んな機能を思いつく限り実装したので、完成までに一年ほどかかってしまいました。

今回は「**ネタバレを防止して交流する**」という一番実現したいことを最優先に制作したので仕事をしながら５ヶ月ほどで完成させることができました。また、メンターの方に進捗管理していただいのも非常にありがたかったです。

2. インプットはそこそこでアウトプットをメインに

完全に理解するのはかなり時間がかかるので、アウトプットメインで学習を行いました。
PHPとLaravelの学習は合わせて三週間くらいで終わらせたと思います。

また、メンターの方に質問する際にどのような状況でエラーが発生するかを分かりやすく説明することを心がけました。メンターの方はただ回答を言うだけでなく、どのように調べれば解決に繋がるかを考えてインプットに繋がる回答をしてくださったので、自分の成長を感じながら作成できました。

今後も上記２点のことを意識して学習を進めていきたいと思います。
